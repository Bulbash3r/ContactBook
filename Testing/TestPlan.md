### 1. Введение
  Содержание данного документа описывает план тестирования разрабатываемого приложения "Contact Book". Цель проведения тестирования - проверка приложения в соответствии с SRS.

<a name="2"></a>
### 2. Объект тестирования
Объектом тестирования является приложение, назначение которого - запись номеров телефонов и, в соответствии с назначением, оно должно обладать определенным функционалом. Приложение должно обладать следующими атрибутами качества: 
   
   1. Функциональность:
+ функциональная полнота: приложение должно выполнять все заявленные функции в соответствии с SRS;
+ функциональная корректность: приложение должно выполнять все заявленные функции корректно;
+ функциональная целесообразность: отсутствуют незаявленные функции, которые бы мешали приложению выполнять первоначально поставленные задачи.

2. Удобство использования:
+ эстетика пользовательского интерфейса: элементы управления объектами должны быть всегда доступны пользователю;
+ актуальность: все изменения происходят в режиме реального времени

### 4. Аспекты тестирования
В ходе тестирования должна быть проверена реализация основных функций приложения, к которым относятся:
* возможность добавления нового контакта;
* возможность просмотра уже добавленных контактов;
* возможность изменения информации о контакте;
* возможность поиска контакта;
* возможность удаления контакта из списка;
* возможность сохранения контактов;
* возможность смены языка приложения.

#### Функциональные требования:

##### Возможность добавления нового контакта
Этот вариант использования небходимо протестировать на:
1. Ввод корректных данных контакта.
2. Ввод некорректных данных контакта.

##### Возможность изменения данных контакта
Этот вариант использования небходимо протестировать на:
1. Ввод корректных данных контакта.
2. Ввод некорректных данных контакта.

##### Возможность удаления контакта из списка
Этот вариант использования небходимо протестировать на корректность удаления контакта при нажатии на соответсвующую кнопку.

##### Возможность просмотра уже добавленных контактов
Этот вариант использования небходимо протестировать на корректность списка отображаемых контактов.

##### Возможность смены языка приложения
Этот вариант использования небходимо протестировать на корректность смены языков и качество локализации.

##### Возможность сохранения контактов
Этот вариант использования небходимо протестировать на корректность загрузки списка контактов после запуска приложения.

#### Нефункциональные требования:
1. Интуитивно-понятный интерфейс.
2. Крупные элементы управления.
3. Возможность удалять добавленные по ошибке контакты.
4. Проверки на правильность ввода и заполненность всех полей.
